üîÑ Auto-Restart Capabilities

This solution provides intelligent automatic restart functionality for Azure App Services:

  • Real-time detection of App Service stop events via Activity Log alerts
  • Tag-based protection - only restarts services marked with "AlwaysOn: true"
  • Immediate email notifications when restart actions are taken
  • Managed Identity for secure Azure API access without storing credentials
  • Logic App conditional logic prevents unnecessary restarts
  • Works for manual stops, scale-down events, and maintenance windows
  • Comprehensive logging and monitoring of all restart actions
  • Parallel execution of notifications and restart logic

‚ö° Logic App Workflow

The Logic App implements a sophisticated conditional restart process:

1. Trigger Reception: Receives webhook from Activity Log Alert containing stop event details
2. Data Parsing: Extracts resource ID, operation name, timestamp, and app name from alert payload
3. Tag Validation: Queries Azure REST API to retrieve App Service tags using Managed Identity
4. Conditional Check: Verifies if "AlwaysOn" tag exists and equals "true"
5. Restart Action: If tag validation passes, sends start command to App Service via Azure REST API
6. Status Response: Returns success/failure status with detailed information for monitoring
