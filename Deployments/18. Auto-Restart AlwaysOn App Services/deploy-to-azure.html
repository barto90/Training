üõ†Ô∏è Post-Deployment Setup

Step 1: Verify Deployment

After deployment, verify all resources are created successfully:

1. Go to Azure Portal ‚Üí Resource Groups ‚Üí [Your Resource Group]
2. Verify these resources exist:
   - App Service with "AlwaysOn: true" tag
   - Logic App with system-assigned managed identity
   - Activity Log Alert targeting your App Service
   - Action Group with email and Logic App actions
   - Log Analytics Workspace
   - Application Insights instance

Step 2: Test Auto-Restart Functionality

Test the solution by stopping your App Service:

# Using Azure CLI
az webapp stop --name <app-service-name> --resource-group <resource-group>

# Using PowerShell
Stop-AzWebApp -Name "<app-service-name>" -ResourceGroupName "<resource-group>"

# Or use the included test script
.\test-auto-restart.ps1 -AppServiceName "myapp" -ResourceGroupName "myResourceGroup"

Step 3: Monitor Activity

Monitor the auto-restart process:

1. Check Email: You should receive an email notification within 1-2 minutes
2. Azure Portal: Go to Logic App ‚Üí Runs History to see execution details
3. Log Analytics: Query activity logs for restart events
4. App Service: Verify the service automatically restarted
‚ö†Ô∏è Important Notes:
  • The App Service name must be globally unique across Azure
  • Deployment requires Contributor role permissions in the target resource group
  • The solution creates managed identities and role assignments automatically
  • Only App Services tagged with AlwaysOn: true will be automatically restarted
  • Email notifications are sent for all restart actions taken
